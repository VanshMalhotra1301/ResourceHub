Matrix Chain Multiplication

Time: O(N³) Space: O(N²) Dynamic Programming

Given a sequence of matrices, find the most efficient way to multiply them together. The problem is not actually to perform the multiplications, but merely to decide the order. We solve this by finding the cost of chains of length 1, then length 2, and so on, building up to the full chain diagonally across the DP table.

DP Strategy
function MatrixChainOrder(p):
n ← length(p) - 1
m ← matrix(n, n), s ← matrix(n, n)
for i ← 1 to n: // Cost of 1 matrix is 0
m[i][i] ← 0
for L ← 2 to n: // Chain length L
for i ← 1 to n - L + 1:
j ← i + L - 1
m[i][j] ← ∞
for k ← i to j - 1: // Test split at k
q ← m[i][k] + m[k+1][j] + (p[i-1]*p[k]*p[j])
if q < m[i][j]:
m[i][j] ← q
s[i][j] ← k // Save split point
return m[1][n], s
