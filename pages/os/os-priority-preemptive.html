Priority Scheduling (Preemptive)

In the preemptive version, the OS actively monitors the Ready Queue. If a new process arrives with a higher priority (lower number) than the currently running process, the CPU is immediately preempted and given to the new arrival.

Preemptive Priority Theory

How it Works

The scheduler evaluates the Ready Queue every time a new process arrives or a process completes. If a newly arrived process has a lower priority number (meaning higher priority) than the running process, a Context Switch occurs.

Tie-Breaker Logic

If a new process arrives with the exact same priority as the currently running process, the OS typically uses FCFS. To minimize context switch overhead, the currently running process is allowed to continue.

Starvation & Context Overhead

Preemptive Priority suffers from both Starvation (low priority jobs never run) and high Context Switch overhead if priorities frequently change or many high-priority jobs arrive rapidly. Aging is still required to solve starvation.
