Prim's Minimum Spanning Tree

Time: O(E log V) Space: O(V + E) Greedy Strategy

Prim's algorithm finds the Minimum Spanning Tree (MST) of a weighted undirected graph. It starts from an arbitrary node and greedily grows the spanning tree one edge at a time. At each step, it selects the minimum weight edge that connects a visited node to an unvisited node.

Prim's Algorithm Logic
function PrimMST(Graph):
visited ← empty set
MST ← empty list
PQ ← MinPriorityQueue()
// Start from an arbitrary node (e.g., node A)
visited.add(A)
for each edge (A, v, weight) in Graph:
PQ.insert(A, v, weight)
while PQ is not empty and visited.size < V:
// Greedily pick the minimum weight edge
(u, v, weight) ← PQ.extractMin()
if v in visited:
continue // Avoid cycles
MST.append((u, v, weight))
visited.add(v)
// Add new frontier edges to PQ
for each edge (v, next, w) in Graph:
if next not in visited:
PQ.insert(v, next, w)
return MST

DAA Exam & Theory Hub

Definition

Prim's Algorithm is a greedy algorithm that finds a Minimum Spanning Tree (MST) for a weighted undirected graph. This means it finds a subset of the edges that forms a tree that includes every vertex, where the total weight of all the edges in the tree is minimized.

Working of the Algorithm

The algorithm operates by building this tree one vertex at a time, starting from an arbitrary initial vertex:

  1. Initialize: Create a set visited to keep track of vertices already included in the MST. Choose an arbitrary starting node and add it to visited.
  2. Frontier Edges: Keep track of all edges that connect a node in the visited set to a node outside the visited set. (Typically done using a Min-Priority Queue).
  3. Greedy Choice: Extract the edge with the absolute minimum weight from the priority queue.
  4. Validation: Check if the destination node of this edge is already in the visited set.
    • If YES: Discard the edge (adding it would create a cycle).
    • If NO: Add the edge to your MST, add the new node to visited, and push all edges originating from this new node into the priority queue.
  5. Repeat: Continue steps 3 and 4 until the visited set contains all $V$ vertices.

Complexity Analysis

  • Time Complexity: O(E log V) - Where $E$ is the number of edges and $V$ is the number of vertices. Each edge is added to the Min-Heap once. Extracting the minimum element from a heap of size $E$ takes $O(\log E)$ which is equivalent to $O(\log V)$.
  • Space Complexity: O(V + E) - $O(V)$ for the visited array and MST storage, and $O(E)$ for the priority queue in the worst-case dense graph.

Q: Explain Prim's Algorithm for finding a Minimum Spanning Tree. (5 Marks)

1. Definition:

Prim's algorithm is a greedy approach to find the Minimum Spanning Tree (MST) of a connected, undirected, and weighted graph. An MST connects all vertices without any cycles and with the minimum possible total edge weight.

2. Algorithm Steps:

  • Start with an empty spanning tree and a set of unvisited vertices.
  • Select an arbitrary vertex and mark it as visited.
  • Find the edge with the minimum weight that connects a visited vertex to an unvisited vertex.
  • Add this edge and the new vertex to the MST. Mark the vertex as visited.
  • Repeat the process until all vertices are visited.

3. Complexity:

Using a binary min-heap and adjacency list, the time complexity is O(E log V).

Q: Write Prim's algorithm. Trace it on a sample graph and analyze its time complexity. (10 Marks)

1. Introduction:

Prim's is a greedy algorithm based on the Cut Property. It maintains two sets of vertices: those included in the MST and those not yet included. At every step, it considers the "cut" separating the two sets and picks the minimum weight crossing edge.

2. Algorithm (Pseudocode):

PRIM(G, w, r) // G=Graph, w=weights, r=root node 1. For each u ∈ G.V: 2. u.key = ∞ 3. u.π = NIL 4. r.key = 0 5. Q = G.V // Min-Priority Queue 6. While Q ≠ Ø: 7. u = EXTRACT-MIN(Q) 8. For each v ∈ G.Adj[u]: 9. If v ∈ Q and w(u,v) < v.key: 10. v.π = u 11. v.key = w(u,v)

3. Complexity Analysis:

The time complexity depends heavily on the data structure used for the Priority Queue Q.

  • Initialization: Takes $O(V)$.
  • Building the Priority Queue: Takes $O(V)$.
  • EXTRACT-MIN loop: Executes $V$ times. Using a binary heap, this takes $O(V \log V)$.
  • Edge relaxation loop: Executes $E$ times overall (once for each edge). Updating keys in a binary heap takes $O(\log V)$. Total time is $O(E \log V)$.

Total Time Complexity: $O(V \log V + E \log V) \implies \mathbf{O(E \log V)}$.

The Cut Property

Prim's algorithm is mathematically backed by the Cut Property of graphs. Imagine slicing a graph into two disjoint sets of vertices (a "cut").

The cut property states that for any cut, the edge with the minimum weight that crosses the cut (connects a node in Set A to a node in Set B) must belong to the Minimum Spanning Tree.

How Prim's uses this:

In Prim's algorithm, Set A is the visited nodes (the growing MST), and Set B is the unvisited nodes. The priority queue stores all the edges "crossing the cut". By constantly extracting the minimum edge from the queue, we are repeatedly applying the Cut Property to guarantee an optimal MST.

The biggest variation to MST generation is choosing between Prim's and Kruskal's algorithm depending on the graph's structure.

Prim's Algorithm

Growth: Vertex-based. Grows a single tree outwards.

Best For: Dense graphs (where $E$ is close to $V^2$). If implemented with an Adjacency Matrix, it runs in $O(V^2)$, which is faster than Kruskal's $O(E \log E)$ for very dense graphs.

Kruskal's Algorithm

Growth: Edge-based. Sorts all edges globally and merges a forest of trees using Disjoint Set Union (DSU).

Best For: Sparse graphs. Running time is bounded by the edge sorting: $O(E \log E)$.

Quick Revision Flashcards

  • Concept: Find MST by growing a single tree from a starting vertex.
  • Strategy: Greedy. Always pick the lowest-weight edge connecting the tree to a new node.
  • Data Structures: Adjacency List, Visited Array (Boolean), Min-Priority Queue.
  • Cycle Prevention: Ignore an edge if its destination node is already marked `visited = true`.
  • Time Complexity: $O(E \log V)$ with adjacency list and binary heap.
  • Space Complexity: $O(V)$ auxiliary space.
  • Comparison: Prim's is better for Dense graphs; Kruskal's is better for Sparse graphs.
