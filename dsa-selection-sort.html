Selection Sort

Best: O(n²) Avg / Worst: O(n²) Space: O(1) In-Place Sorting

Selection Sort divides the array into a sorted and an unsorted region. It repeatedly scans the unsorted region to find the minimum element, and swaps it with the first element of the unsorted region, expanding the sorted boundary by one.

Pseudocode
function selectionSort(arr):
n ← len(arr)
for i ← 0 to n - 2:
min_idx ← i ← assume first is min
for j ← i + 1 to n - 1: ← scan rest
if arr[j] < arr[min_idx]:
min_idx ← j ← new minimum found
if min_idx != i:
swap(arr[i], arr[min_idx]) ← move min to front ✅

Complexity Analysis
Best Case Time
O(n²)
Even if sorted, it scans the entire remaining array to find the minimum.
Average Time
O(n²)
Comparisons are n(n-1)/2 regardless of initial order.
Worst Case Time
O(n²)
Reverse sorted arrays yield max comparisons and swaps.
Space
O(1)
In-place sorting; only a few variables are used.
How Selection Sort Works
1
Setup: The array is divided into two parts: sorted (left) and unsorted (right). Initially, sorted is empty.
2
Set pointer i at the start of the unsorted part. Assume the element at i is the minimum.
3
Use pointer j to scan the rest of the unsorted array to the right.
4
If an element smaller than the current minimum is found, update the minimum's index.
5
Once the scan is complete, swap the found minimum with the element at i. The sorted part grows by 1.
