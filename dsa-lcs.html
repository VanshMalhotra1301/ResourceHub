Longest Common Subsequence

Time: O(m × n) Space: O(m × n) Dynamic Programming

The LCS problem finds the longest subsequence present in both strings. It uses a 2D matrix to build overlapping subproblems. If characters match, we add 1 to the diagonal value. If they don't, we take the maximum of the top or left cell. Finally, we backtrack to reconstruct the string.

DP Strategy & Backtracking
function LCS(A, B):
m ← len(A), n ← len(B)
dp ← grid of size (m+1) x (n+1) filled with 0
for i from 1 to m:
for j from 1 to n:
if A[i-1] == B[j-1]: // Characters Match
dp[i][j] = dp[i-1][j-1] + 1 // Diagonal + 1
else: // No Match
dp[i][j] = max(dp[i-1][j], dp[i][j-1]) // Top or Left
// Backtracking from dp[m][n]
i ← m, j ← n, lcs_str ← ""
while i > 0 and j > 0:
if A[i-1] == B[j-1]:
lcs_str.prepend(A[i-1])
i--, j--
else if dp[i-1][j] > dp[i][j-1]:
i--
else:
j--
return lcs_str

DAA Exam & Theory Hub

Definition

The Longest Common Subsequence (LCS) problem is a classic dynamic programming challenge where the goal is to find the longest subsequence common to all sequences in a set of sequences (usually just two). A subsequence is a sequence that can be derived from another sequence by deleting some elements without changing the order of the remaining elements. (e.g., "ACE" is a subsequence of "ABCDE").

Working of the Algorithm

Instead of generating all possible subsequences ($2^n$ combinations) and comparing them, we break the problem down using a 2D Matrix:

  1. Initialization: Create a 2D array DP[m+1][n+1] where $m$ and $n$ are lengths of strings X and Y. Fill the first row and first column with 0s (Base Case: comparing with an empty string yields 0 length).
  2. Matrix Filling: Iterate through the characters of both strings.
    • If characters match: Add 1 to the result of the sequences without these current characters (diagonal value).
    • If they do not match: Take the maximum of the LCS formed by excluding the current character of X, or excluding the current character of Y (max of top or left cell).
  3. Backtracking (Reconstruction): Start from the bottom-right corner DP[m][n]. If characters match, include it in the LCS and move diagonally up-left. If they don't, move in the direction of the larger value (top or left) until you hit a 0.

Recurrence Relation

LCS(X[1..i], Y[1..j]) =

  0                           if i = 0 or j = 0
  LCS(X[1..i-1], Y[1..j-1]) + 1    if X[i] == Y[j]
  max( LCS(X[1..i-1], Y[j]),
       LCS(X[1..i], Y[j-1]) )      if X[i] != Y[j]

Complexity Analysis

  • Time Complexity: O(m × n) - We use two nested loops to fill a table of size $(m+1) \times (n+1)$. Each table entry takes $O(1)$ time to compute.
  • Space Complexity: O(m × n) - Required to store the DP table. Note: Space can be optimized to $O(\min(m,n))$ if we only need the length, by just keeping two rows in memory.

Q: Explain the Longest Common Subsequence (LCS) problem and write its recurrence relation. (5 Marks)

1. Definition:

A subsequence is a sequence derived from another by deleting elements without changing the order. The LCS problem aims to find the longest sequence present in both given strings.

2. Approach:

It is solved efficiently using Dynamic Programming because it possesses the Optimal Substructure and Overlapping Subproblems properties. We build a bottom-up 2D table to avoid redundant calculations.

3. Recurrence Relation:

If X[i] == Y[j]: DP[i][j] = DP[i-1][j-1] + 1
If X[i] != Y[j]: DP[i][j] = max(DP[i-1][j], DP[i][j-1])

Q: Define LCS. Write the algorithm to find the length of LCS and analyze its complexity. (10 Marks)

1. Introduction & Definition:

Given two sequences, find the length of longest subsequence present in both. It is widely used in DNA sequence alignment and file comparison (like 'diff' tools).

2. Dynamic Programming Formulation:

  • State Definition: Let DP[i][j] be the length of LCS of prefix X[0...i-1] and Y[0...j-1].
  • Base Case: DP[i][0] = 0 and DP[0][j] = 0 for all $i, j$.

3. Algorithm:

LCS-LENGTH(X, Y) 1. m = length(X), n = length(Y) 2. Let DP[0..m, 0..n] be a new table 3. For i = 0 to m: DP[i, 0] = 0 4. For j = 0 to n: DP[0, j] = 0 5. For i = 1 to m: 6. For j = 1 to n: 7. if X[i] == Y[j]: 8. DP[i, j] = DP[i-1, j-1] + 1 9. else: 10. DP[i, j] = max(DP[i-1, j], DP[i, j-1]) 11. Return DP[m, n]

4. Complexity Analysis:

Time Complexity: The algorithm contains nested loops iterating $m$ and $n$ times. Inside the loop, constant time operations are performed. Thus, Time Complexity is O(m × n).

Space Complexity: The 2D table requires $(m+1) \times (n+1)$ memory allocations. Thus, Space Complexity is O(m × n).

Why Does DP Work Here?

To use Dynamic Programming, a problem must satisfy two strict conditions. LCS perfectly demonstrates both:

1. Optimal Substructure

A problem has optimal substructure if an optimal solution can be constructed from optimal solutions of its subproblems. In LCS, if the last characters of X and Y match, the LCS must end with this character. The problem immediately reduces to finding the LCS of the remaining prefixes. If they don't match, the LCS is simply the longest found by either dropping the last character of X or Y.

2. Overlapping Subproblems

If you implement LCS using pure recursion, you will solve the exact same subproblem multiple times. By using the DP matrix, we memoize (or tabulate) the results. When we need the LCS of prefixes of length $i$ and $j$, we just look it up in DP[i][j] in $O(1)$ time instead of recalculating it.

The LCS algorithm is a foundation. By slightly tweaking the recurrence relation or the backtracking logic, you can solve many famous string processing problems:

Longest Common Substring

Unlike subsequences, substrings must be contiguous. The DP is similar, but if X[i] != Y[j], we reset DP[i][j] = 0 instead of taking the maximum.

Edit Distance (Levenshtein)

Finds minimum operations (insert, delete, replace) to convert string A to B. It uses a very similar DP matrix but tracks penalty costs instead of string lengths.

Shortest Common Supersequence

Finds the shortest string that has both A and B as subsequences. Length is simply: Length(A) + Length(B) - LCS(A,B).

Quick Revision Flashcards

  • Core Concept: Subsequence = characters in order, but not necessarily consecutive.
  • Data Structure: 2D Matrix of size (m+1) × (n+1).
  • Base Case: Row 0 and Col 0 are entirely 0.
  • Match Condition: Diagonal value + 1.
  • Mismatch Condition: Max of Top OR Left value.
  • Backtracking: Start at bottom right. Move diagonal if match, otherwise move towards the larger adjacent number.
  • Time Complexity: O(m × n).
  • Space Complexity: O(m × n) standard, O(min(m,n)) optimized.
