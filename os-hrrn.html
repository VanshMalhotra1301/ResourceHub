Highest Response Ratio Next (HRRN)

HRRN is a non-preemptive algorithm designed to solve the starvation problem of SJF. It dynamically calculates a priority based on how long a process has been waiting. The longer a process waits, the higher its Response Ratio becomes, guaranteeing it will eventually run.

The Response Ratio Formula

The decision of which process to execute next is made when the CPU becomes idle. The OS calculates the ratio for every waiting process:

RR = (Waiting Time + Burst Time) / Burst Time
RR = 1 + (W / S)

Key Advantages

  • Prevents Starvation: Because Waiting Time (W) is in the numerator, the longer a job waits, the higher its ratio climbs. It will eventually out-prioritize even the shortest incoming jobs.
  • Favors Shorter Jobs initially: If waiting time is 0, RR is 1 for all processes. As time passes, the ratio grows much faster for processes with a small Burst Time (S).
  • Mode: Non-Preemptive. Once it grabs the CPU, it runs to completion.
